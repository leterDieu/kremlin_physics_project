# запись условия
> - в "Дано:" у задач всегда нужно указывать модуль; ответ, чаще всего, тоже даётся с модулем
>> [!abstract] одинаковых букв (даже с разными индексами) у разных понятий быть не должно: 
>> + T - период, секунды
>> + T - температура, градусы Кельвина
>> или 
>> + t - время, секунды
>> + t - температура, градусы Цельсия
> - в задаче должно быть в конце написано "Ответ"
---
# вычисления
> - промежуточные вычисления **допустимы**, однако снижают лояльность проверяющего эксперта. В сложных задачах и оптике - приветствуется (иногда)
> - в подсчёте по формуле не обязательно прописывать единицы измерения после каждого преобразования, т.е.:
>> $W = \frac{Cu^2}{2} = \frac{100\text{Ф} \cdot 10^2\text{В}^2}{2} = \frac{100 \cdot 10^2}{2} = 5000\text{ Дж}$
>> note: указывать единицы измерения в начале и в конце - обязательно.
---
# рисунки
> - почти для любой задачи следует нарисовать схему. Это требуется как для того, чтобы **показать действия сил**, расставить **скорости и ускорения** (и проч.), так и для **обозначения осей (Ox, Oy)**.
> - в оптике рисунок должен быть не "прикидкой", а полным и точным отражением задачи - рекомендуется рисовать достаточно большое изображение происходящего (лучше всего первый вариант на черновике)
> - для величин (таких как например силы), не упомянутых в условии, требуется оформление в формате "пусть Fтр1 - сила трения между телом 1 и телом 2"; однако для большинства из них можно просто показать на рисунке силу (или другую величину, скорость например) и в индексе подписать характер (Fтр - стандартное для силы трения).
